## I{NN}-1. ワークフロー中間レビュー

### 背景

- Method B ワークフローの定期的な振り返りとして実施する
- 前回レビュー以降のイシュー運用を評価し、改善点を特定する

### 要求

以下の評価を実施すること:

1. **対象イシューの評価**: 下記「評価対象」に記載されたイシューを11軸×5段階で評価する
2. **総合評価**: スコア集計、強み・弱みの分析
3. **改善提言**: 具体的な改善提案を優先度付きで提示する
4. **前回提言のフォローアップ**: 前回の改善提言が反映されたかを確認する

### 評価フレームワーク

#### A. 人間側（Issue 設計の質）

| 軸 | 定義 |
|----|------|
| A1. 明確性 | 要求・完了条件が具体的で解釈の余地が少ないか |
| A2. 範囲の適切さ | 1回のイシューで扱うべき分量として適切か |
| A3. 文脈提供 | 背景情報・制約条件が十分に示されているか |
| A4. 段階的発展 | 前回の成果を踏まえた自然な次のステップか |

#### B. AI側（対応の質）

| 軸 | 定義 |
|----|------|
| B1. 完全性 | 全ての要求・完了条件を満たしたか |
| B2. 正確性 | 出力内容が事実に基づき、論理的に正しいか |
| B3. 実用性 | 出力がそのまま活用・参照できるか |
| B4. 自律性 | 明示的な指示を超えた有益な提案・修正を行ったか |

#### C. ワークフロー全体

| 軸 | 定義 |
|----|------|
| C1. 効率性 | 作業が効率的に進行したか（手戻り・冗長の少なさ） |
| C2. 追跡可能性 | 判断の経緯が後から辿れるか |
| C3. 累積的価値 | イシュー群全体として一貫した成果を積み上げているか |

#### 評価基準

| スコア | 意味 |
|--------|------|
| 5 | 優秀: 改善の余地がほとんどない |
| 4 | 良好: 小さな改善点はあるが全体として適切 |
| 3 | 普通: 機能しているが明確な改善点がある |
| 2 | 不十分: 重要な問題があり改善が必要 |
| 1 | 不適切: 目的を達成できていない |
