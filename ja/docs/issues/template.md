# イシュー テンプレート定義

## イシュー番号体系

- **通し番号**: `I{NN}-{n}` 形式（NN: 2桁のイシュー番号、n: サブイシュー番号）
  - 例: `I08-1`, `I08-2`, `I09-1`
- **テーマの区切り**: 異なるテーマは異なるイシュー番号を持つ
  - 例: I08-1, I08-2 が1つのテーマ → 次のテーマは I09-1
- **1ファイル複数テーマ**: `---`（水平線）で区切る。AI は各テーマを順番に処理し、テーマごとに Phase 4 を完了させる
- **テンプレート**: `template_issue_open.md` に `{NN}` プレースホルダがある。コピー時に実際の番号に置換する

### 複数イシューの運用ルール

1. `issue_open.md` を読み取り、イシューの数を確認する
2. 各イシュー（テーマ）ごとに順番に作業する（Phase 4 まで完了して独立させる）
3. 作業途中でエラーが発生した場合: 完了済みイシューはそのまま、次のイシューには入らず終了する

## イシューファイルのスキーマ

### ヘッダ

```yaml
Created: YYYY-MM-DD    # 作成日
Category: [category]   # カテゴリ（空欄可、AI が内容から判断して付与）
```

### 項目

```markdown
## I{NN}-{n}. タイトル

### 背景
[このイシューの動機・文脈]

### 要求
[具体的な作業内容]

### 補足情報

#### 成果物形式
{文書 / スキーマ / コード / 判断 / スクリプト}

#### 関連フォルダ/ファイル
- {リポジトリルートからの相対パス}

#### 関連issue
{例: I07-3, なければ「なし」}

### 完了条件
- [ ] 条件1（3〜5個を目安）
- [ ] 条件2
```

最小構成（背景・要求・完了条件は省略可）:

```markdown
## I01-1. タイトル

本文
```

### 複数テーマの記載

異なるテーマは `---`（水平線）で区切り、イシュー番号を1つ上げる:

```markdown
## I08-1. テーマA

...

---

## I09-1. テーマB

...
```

AI は各テーマを順番に処理し、テーマごとに Phase 4 を完了させる。

### 対応セクション（AI が完了時に追加）

```markdown
### 対応

[対応内容]
```

### 横断的な補足セクション（任意）

複数項目にまたがる補足情報がある場合、最後の項目の後に記載:

```markdown
## 補足

[複数項目に関わる補足情報]
```

## イシュー設計ガイドライン

- **1イシュー1テーマの原則**: サブイシューが3つを超える場合は分割を検討
- **完了条件**: 3〜5個を目安とする。**作業内容のみ**を記載し、Phase 4 の手続き（アーカイブ、close_issue.sh 実行等）は含めない
- **成果物形式の明示**: 期待する成果物の形式（文書 / スキーマ / コード / 判断 / スクリプト）を記載
- **依存関係の宣言**: 前提となるイシューがあれば「関連issue」に明記

## カテゴリラベル

| カテゴリ | 対象 |
|----------|------|
| `verification` | 計算結果・論理の検証 |
| `research` | 調査・方向性提示 |
| `proposal` | 方針・設計の提案 |

| `docs` | ドキュメント全般 |
| `workflow` | ワークフロー・運用方法 |

## 完了ファイル命名規則

```
done/issue_YYMMDD_NN.md
```

- `YYMMDD`: 作成日（6 桁）
- `NN`: 同日内の連番（2 桁ゼロ埋め）

## Phase 4（完了処理）

1. `issue_open.md` の完了条件チェックボックスを `[x]` に更新し、各項目に `### 対応` セクションを追記
2. `docs/issues/done/issue_YYMMDD_NN.md` にコピー
3. `issue_open.md` を `template_issue_open.md` の内容でリセット（`{NN}` を次の番号に置換）
4. `docs/issues/issue_history.md` に記録を追加
5. `docs/issues/artifact_index.md` に成果物一覧を追記
6. `handover/handover_memo_latest.md` を更新
7. Git 完了処理: commit → push → squash merge to main → sync branch

> **スクリプト**: `handover/scripts/close_issue.sh` で自動化可能
> **注意**: 完了条件には作業内容のみを記載する。Phase 4 の手続きステップ（上記1-7）は完了条件に含めない

完了後のブランチ状態:
- **main**: squash commit
- **ai/workflow_issue**: `git reset --hard main && git push --force-with-lease` で main と同期済み
- **issue_open.md**: 両ブランチでテンプレート状態
