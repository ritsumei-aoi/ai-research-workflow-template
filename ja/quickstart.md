# クイックスタートガイド

AI Research Workflow Template を使い始めるためのガイドです。
ご利用の AI 環境に合った方法を選んでください。

> **どの方法を選べばいいかわからない？** ページ下部の [Method Z](#method-z-aiに丸投げする) へどうぞ — どの AI でも使えます。

---

## 前提準備

1. **リポジトリを作成**: このテンプレートからリポジトリを作成（またはクローン/フォーク）
2. **言語を選択**: `en/` または `ja/` ディレクトリ内で作業
3. **[CUSTOMIZE.md](ja/CUSTOMIZE.md)** に従い初期セットアップを実施

---

## Method A: 対話型ワークフロー

**対象:** ブラウザベースの AI チャット（ChatGPT、Claude、Gemini web 等）で、
AI が指示を出し、人間がローカルで変更を適用する方式。

### 必要なもの

- AI チャットインターフェース（ブラウザまたはアプリ）
- このリポジトリのローカルコピー
- テキストエディタとターミナル

### 手順

**ステップ 1: プロジェクトの準備**

[CUSTOMIZE.md](ja/CUSTOMIZE.md) のセットアップを完了し、以下を記入:
- `handover/ai_trust_policy.md` — AI に委任可能な範囲を定義
- `handover/handover_memo_latest.md` — プロジェクトの現在の状態を記述

**ステップ 2: セッション開始**

AI チャットを開き、コンテキストを提供します。方法は:
- **ファイルを添付**（多くのプラットフォームで推奨）
- **内容をプロンプトに貼り付け**

最低限、以下を提供してください:
- `handover/workflow_common.md`
- `handover/workflow_method_a.md`
- `handover/handover_memo_latest.md`

> **オプション:** 一部の AI プラットフォームはツールベースのファイル取得（GitHub リポジトリやクラウドストレージにアクセスする MCP サーバ等）をサポートしており、手動添付の代わりに AI が直接ファイルを読み取ることも可能です。

**ステップ 3: 最初のプロンプト例**

```
研究プロジェクトの新しいセッションを開始します。
ワークフロールールと引き継ぎ資料を添付しました。
内容を読んで、現在の状態の理解を確認してください。

本日は [タスクの説明] に取り組みたいと思います。
```

**ステップ 4: セッション中の作業**

AI が変更を提案し、あなたがローカルで適用、結果を報告してイテレーションします。

**ステップ 5: セッション終了**

チャットを閉じる前に:
1. `handover/handover_memo_latest.md` を実施内容で更新
2. 変更をコミット・プッシュ
3. 更新された引き継ぎメモにより、次回のセッション（異なる AI でも）がシームレスに続行可能

### Method A の流れ

```
あなた（人間）        AI（チャット）
    │                    │
    ├─ ファイル添付 ────►│
    │                    ├─ 読み込み・確認
    │◄── 提案 ──────────┤
    ├─ 変更を適用        │
    ├─ 結果を報告 ──────►│
    │                    ├─ レビュー・イテレーション
    │◄── 次のステップ ──┤
    ├─ 引き継ぎ更新      │
    └─ コミット・プッシュ │
```

## Method B: イシュー駆動エージェント型

**対象:** ファイルシステムに直接アクセスできる AI エージェント（VS Code の GitHub Copilot、
Gemini CLI 等）で、AI がファイルの読み書き・コマンド実行を自律的に行う方式。

### 必要なもの

- ファイルシステムとシェルアクセスを持つ AI エージェント、例えば:
  - **GitHub Copilot**（VS Code / CLI）— エージェントモード
  - **Gemini CLI** — ローカルワークスペースアクセス付き
  - 同等の機能を持つ他の AI コーディングエージェント
- このリポジトリのローカルコピー

### 手順

**ステップ 1: プロジェクトの準備**

[CUSTOMIZE.md](ja/CUSTOMIZE.md) のセットアップを完了し、以下を記入:
- `handover/ai_trust_policy.md`
- `handover/handover_memo_latest.md`

**ステップ 2: イシューを作成**

`docs/issues/issue_open.md` にテンプレートに従ってタスクを記述:

```markdown
Created: 2026-04-06
Category: implementation

## I1. データローダのユニットテスト追加

### 背景
`src/loader.py` のデータローダモジュールにテストカバレッジがない。

### 要求
主要な読み込み関数をカバーする pytest ベースのユニットテストを作成する。

### 完了条件
- [ ] `load_json()` と `load_csv()` 関数のテストがある
- [ ] `pytest tests/ -v` で全テストが通る
```

> 完了済みイシューの実例は [samples/docs/issues/done/](samples/docs/issues/done/) を参照。

**ステップ 3: AI エージェントを起動し最初のプロンプトを入力**

```
新規イシューを作成しました。docs/issues/issue_open.md を読んで、
handover/workflow_method_b.md に定義されたワークフローに従い対応してください。
```

AI エージェントは以下を実行します:
1. ワークフロールールと引き継ぎ資料を読み込み
2. 読み込み確認を出力（ブランチ名、前回の成果、今回の目標）
3. イシュー項目への対応を開始

**ステップ 4: 監視と承認**

AI は自律的に作業しますが、破壊的操作の前には確認を求めます。
進捗を確認し、必要に応じてガイダンスを提供してください。

**ステップ 5: 完了**

AI は Phase 4 完了プロセスを自動的に実行します:
- 完了条件のチェックボックスをマーク
- イシューを `docs/issues/done/` にアーカイブ
- `handover_memo_latest.md` を更新
- main にマージ

> **備考:** このガイドではユーザと AI が同じローカル環境で作業することを前提としています。
> Git ベースの受け渡しも可能です: イシューファイルをコミット・プッシュし、
> AI に別の環境でプルさせて作業させることもできます。

### Method B の流れ

```
あなた（人間）            AI（エージェント）
    │                        │
    ├─ issue_open.md を記述  │
    ├─ エージェント起動 ────►│
    │                        ├─ ワークフロー+引き継ぎ読み込み
    │                        ├─ 読み込み確認を出力
    │                        ├─ イシューに対応
    │◄── 確認要求？ ─────────┤ （破壊的操作の場合）
    ├─ 承認 ────────────────►│
    │                        ├─ 完了・アーカイブ
    │                        └─ 引き継ぎ更新・マージ
    │
    └─ 結果を確認
```

## Method Z: AIに丸投げする

**対象:** ドキュメントを読まずに今すぐ始めたい方。

最もシンプルなアプローチ: お使いの AI（何でも可）にこのリポジトリを渡し、
利用可能なツールでワークフローを使う最適な方法を AI に判断させます。

### ワンプロンプト

以下をお使いの AI チャットにコピー＆ペーストしてください:

```
このAI研究ワークフローテンプレートを自分のプロジェクトで使いたいと思います。
リポジトリはこちらです: https://github.com/ritsumei-aoi/ai-research-workflow-template

私の現在の環境と利用可能なツールに基づいて、以下を提案してください:
1. どの方法（A、B、C）が最適か
2. セットアップと利用開始の具体的な手順
3. 最初のセッションの例

提案の後、セットアップをステップバイステップでガイドしてください。
```

以上です。AI がリポジトリの内容を読み、あなたの環境を評価し、
最適なセットアップを案内してくれます。

### なぜこれが機能するのか

このリポジトリは自己文書化されています: 詳細なワークフロー定義、比較ガイド、
カスタマイズ手順、サンプルイシューが含まれています。
有能な AI であればこれらを読み、あなたの状況に合わせたガイダンスを提供できます。

> **ヒント:** AI が URL に直接アクセスできない場合は、リポジトリをダウンロード/クローンし、
> 主要ファイル（`README.md`、`ja/handover/workflow_methods_comparison.md`、
> `ja/CUSTOMIZE.md`）をチャットに添付してください。

## 次のステップ

最初のセッションを終えると、以下が揃います:
- カスタマイズ済みの `ai_trust_policy.md` を持つプロジェクト
- プロジェクト状態を記録する `handover_memo_latest.md`
- イシュー → 対応 → アーカイブのサイクルへの理解

より深い理解のために:
- [Method 比較ガイド](ja/handover/workflow_methods_comparison.md) — 全方式の詳細比較
- [共通ルール](ja/handover/workflow_common.md) — ブランチ、引き継ぎ、セッションポリシー
- [サンプルイシュー](samples/docs/issues/done/) — 完了済みイシューの実例
