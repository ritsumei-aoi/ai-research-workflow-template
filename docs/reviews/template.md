# レビュースキーマ定義

> レビューファイルのフォーマット仕様。`review_open.md` の雛形として使用する。

---

## 運用フロー

```
1. ユーザが template.md をコピーして review_open.md を作成
2. AI が review_open.md の存在を検出 → 対応開始
3. 対応完了 → review_YYMMDD_NN.md にリネームして done/ へ移動
4. 差し戻し → review_needs_clarification.md にリネーム
```

### ファイル状態

| ファイル名 | 状態 | 意味 |
|-----------|------|------|
| `review_open.md` | 未対応 | AI が対応すべきレビューが存在する |
| `review_needs_clarification.md` | 差し戻し | ユーザの追加指示待ち（最優先） |
| `done/review_YYMMDD_NN.md` | 完了 | 対応済みレビュー（アーカイブ） |
| なし | アイドル | 対応すべきレビューなし |

### AI の自動補完ルール

- `Created: today` → AI が実際の日付（`YYYY/MM/DD`）に置換
- `Category:` 未記入 → AI が内容から判断してラベルを付与
- `Priority:` 省略 → `normal` として扱う
- 種別タグ省略 → AI が内容から適切なタグを推定

## スキーマ定義

### 完了ファイル命名規則

```
review_YYMMDD_NN.md
```

- `YYMMDD`: 作成日（6 桁）
- `NN`: 同日内の連番（2 桁ゼロ埋め）
- 完了後に `done/` へ移動

### ヘッダ（必須）

| フィールド   | 形式               | 説明                                       |
|-------------|--------------------|--------------------------------------------|
| `Created`   | `YYYY/MM/DD`       | レビュー作成日（`today` 可→AI が補完）        |
| `Answered`  | `YYYY/MM/DD` or `—`| 全項目対応完了日（未完了は `—`）               |
| `Status`    | 下記参照            | レビュー全体の状態                            |
| `Category`  | カンマ区切りラベル   | `latex`, `code`, `theory`, `workflow`, `docs` 等 |
| `Priority`  | `high` / `normal` / `low` | 対応の優先度（省略時は `normal`）       |
| `Note`      | 自由記述（任意）     | 前提条件・依存関係・補足                      |

#### Status の値

| 値                  | 意味                                           |
|--------------------|------------------------------------------------|
| `open`             | 未対応（デフォルト）                               |
| `needs_clarification` | 差し戻し：指示不明確のため確認待ち                |
| `done`             | 全項目対応済み                                    |

### 項目（Item）

```markdown
## I{番号}. [{種別タグ}] {タイトル}

{本文}

### R{番号}. 回答

{対応内容}
```

### 種別タグ一覧

| タグ     | 用途                                         |
|---------|----------------------------------------------|
| `質問`   | 確認・説明を求める（回答は説明・根拠）           |
| `指示`   | 作業の実行を求める（回答は実施内容の報告）        |
| `修正`   | 既存内容の修正を求める（回答は修正内容と差分）     |
| `提案`   | 改善案の検討を求める（回答は採否と理由）          |
| `確認`   | 事実・整合性の確認を求める（回答は確認結果）       |

> タグは拡張可能。種別タグの省略も許可（AI が推定）。

### 項目間の区切り

項目間は水平線 (`---`) で区切る。

---

## テンプレート本体

以下をコピーして `review_open.md` を作成する。

```markdown
Created: today
Category:

## I1. {タイトル}

{本文}

---

## I2. {タイトル}

{本文}
```

**最小構成**: `Created` と1つ以上の `## I{N}.` があれば有効。
回答セクション (`### R{N}.`)、`Answered`、`Status`、`Priority`、`Note` は AI が対応時に追加する。
