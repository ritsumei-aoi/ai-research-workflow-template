## I1. コード実行例の Web 提示方法の検討

### 背景

論文で使用するコードは `osp-triviality` リポジトリで公開されている。
論文の読者がインタラクティブな計算を体験できるよう、
Web 上での提示方法を検討する必要がある。
管理者権限を持つ Web サーバが利用可能。

### 要求

コード実行例を Web 上で提示する方式を調査し、
利点・欠点・推奨度を含む比較表を提示すること。
実装は不要（案の提示のみ）。

### 完了条件

- [x] 3 つ以上の方式が比較されている
- [x] 各方式にセットアップ負荷・インタラクティブ性・維持コストの評価がある
- [x] 推奨構成（Phase 1 / Phase 2）が明記されている

---

### 対応

6 方式を比較:

| # | 方式 | セットアップ | インタラクティブ性 | 維持コスト | 推奨度 |
|---|------|------------|-------------------|-----------|--------|
| A | Jupyter Book | 低 | なし | 低 | ★★★ |
| B | Binder | 低 | 高 | 低（外部依存） | ★★★★ |
| C | JupyterHub on 自サーバ | 中〜高 | 最高 | 高 | ★★★ |
| D | Streamlit / Gradio | 中 | 中 | 中 | ★★ |
| E | GitHub Codespaces | 低 | 高 | 有料 | ★★ |
| F | JupyterLite (WASM) | 中 | 高 | 低 | ★★★★ |

**推奨構成**:
- Phase 1（即時）: Binder badge + デモ Notebook（`notebooks/demo_certificate_verification.ipynb`）
- Phase 2（中期）: JupyterLite で静的デモを自サーバに配置

### 検証方法

人間による方針決定（AI 委任境界外：公開戦略の決定は人間が判断）。
