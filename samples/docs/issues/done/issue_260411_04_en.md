## I04-1. MathQForm Schema Definition and Sample Data Creation

### Background

A previous issue proposed MathQForm as a new benchmark format for this research.
Here, we establish a formal specification based on JSON Schema (2020-12) and create sample data.
Text fields use Japanese as the primary language, with a multilingual-ready structure.

### Requirements

- Create schema definition files in `docs/schema/`
- Create bundle-format sample data in `data/samples/`

### Completion Criteria

- [x] Define 4 entities + bundle in JSON Schema
- [x] Introduce multilingual text fields (`{ja, en}` structure)
- [x] Create bundle-format sample data

---

### Response

Created 5 schema files:

| File | Entity | Description |
|------|--------|-------------|
| `mathqform_item.schema.json` | Item | Source memo (informal mathematical notes) |
| `mathqform_question.schema.json` | CandidateQuestion | Structured question candidates |
| `mathqform_annotation.schema.json` | Annotation | Individual evaluation (4 primary + 2 auxiliary axes) |
| `mathqform_adjudication.schema.json` | Adjudication | Consensus judgment |
| `mathqform_bundle.schema.json` | Bundle | Bundled format containing all entities |

**Design highlights**:
- Text fields use `{ "ja": "...", "en": "..." }` format
- ID scheme: `{field}-{seq}-{generator}-{seq}` (e.g., `RA-001-H-01`)
- Evaluation axes constrained to `integer` (1–5)
- `additionalProperties: false` prevents unintended field inclusion

Sample data created as a bundle containing 1 Item each from 2 fields (real_analysis: B4, abstract_algebra: M1).
