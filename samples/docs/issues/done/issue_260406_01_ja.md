## I1. Family I/II Certificate の n 不変性の検証

### 背景

`docs/theory/triviality_certificates.md` によると、Family I・II の自明性 certificate は
n 不変性を満たす: certificate ベクトルは代数のランク n に依存しない（n ≥ n₀ において）。

コード公開レポジトリ（[osp-triviality](https://github.com/ritsumei-aoi/osp-triviality)）には
この性質を検証する `scripts/verify_n_invariance.py` が提供されている。
n 不変性が成り立てば、小さい n で検証した単一の certificate が
すべての n に対する結果を証明するため、自明性証明の鍵となるステップである。

### 要求

1. `docs/theory/triviality_certificates.md` を読み、n 不変性の主張を確認する
2. osp-triviality リポジトリの `scripts/verify_n_invariance.py` を
   B(0,1) から B(0,3) に対して実行し、certificate がランク間で一致することを確認する
3. どのファミリー・ウェイトタイプがカバーされるかをまとめる

### 完了条件

- [x] Family I について n 不変性が確認されている
- [x] Family II について n 不変性が確認されている
- [x] B(0,1)、B(0,2)、B(0,3) の結果が比較・報告されている

---

### 対応

Family I・II について B(0,1)–B(0,3) にわたる検証を完了:

1. **Family I（±δ_k ウェイト）**:
   B(0,1)、B(0,2)、B(0,3) の certificate を抽出。
   ±δ₁ ウェイトの certificate ベクトルは3つのランクすべてで同一。
   B(0,2)・B(0,3) では ±δ₂ の certificate も一致。
   **n 不変性を確認。**

2. **Family II（±δ_k ± δ_l ウェイト）**:
   n ≥ 2 で適用可能（2つ以上の奇ルート対が必要）。
   B(0,2)・B(0,3) の δ₁+δ₂ ウェイトの certificate は同一。
   B(0,3) では追加ウェイト δ₁+δ₃、δ₂+δ₃ の certificate もパターンと整合。
   **n 不変性を確認。**

3. **スクリプト出力**（抜粋）:
   ```
   $ python scripts/verify_n_invariance.py
   Family I:  B(0,1) ✓  B(0,2) ✓  B(0,3) ✓  — certificates match
   Family II: B(0,2) ✓  B(0,3) ✓  — certificates match
   n-invariance: CONFIRMED for Families I and II
   ```

4. **カバー範囲のまとめ**:
   | ファミリー | カバーされるウェイト | n₀ | 状態 |
   |-----------|-------------------|-----|------|
   | I | ±δ_k | 1 | ✓ 確認済み |
   | II | ±δ_k ± δ_l | 2 | ✓ 確認済み |
   | III | ±2δ_k | — | 対象外（設計上ランク依存） |

### 検証方法

自動スクリプト実行（`scripts/verify_n_invariance.py`）— 再現可能。
相互参照: `docs/theory/triviality_certificates.md`「Family I・II: n 不変性」節。
