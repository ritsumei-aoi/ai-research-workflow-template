## I04-1. MathQForm スキーマ設定・サンプルデータ作成

### 背景

前回のイシューにて、本研究で新規に用いるベンチマーク形式 MathQForm が提案された。
ここでは、JSON Schema (2020-12) に基づく正式な規格を定め、サンプルデータを作成する。
テキストフィールドは日本語を主言語としつつ、多言語対応の構造とする。

### 要求

- `docs/schema/` にスキーマ定義ファイルを作成する
- `data/samples/` にバンドル形式のサンプルデータを作成する

### 完了条件

- [x] JSON Schema で4エンティティ＋バンドルを定義する
- [x] 多言語対応テキストフィールド（`{ja, en}` 構造）を導入する
- [x] バンドル形式のサンプルデータを作成する

---

### 対応

5つのスキーマファイルを作成:

| ファイル | エンティティ | 説明 |
|---------|------------|------|
| `mathqform_item.schema.json` | Item | ソースメモ（非形式的な数学メモ） |
| `mathqform_question.schema.json` | CandidateQuestion | 構造化問い候補 |
| `mathqform_annotation.schema.json` | Annotation | 個別評価（4主軸＋2補助軸） |
| `mathqform_adjudication.schema.json` | Adjudication | 合意判定 |
| `mathqform_bundle.schema.json` | Bundle | 全エンティティを束ねたバンドル形式 |

**設計上のポイント**:
- テキストフィールドは `{ "ja": "...", "en": "..." }` 形式
- ID体系: `{field}-{seq}-{generator}-{seq}`（例: `RA-001-H-01`）
- 評価軸は `integer` (1–5) で型制約
- `additionalProperties: false` で意図しないフィールドの混入を防止

サンプルデータは2分野（real_analysis: B4, abstract_algebra: M1）から各1 Item を含むバンドルとして作成。
